1. **Use atomic writes**: Write to a temporary file first, then rename:
   ```python

   with h5py.File(temp_file, 'w') as f:

   os.rename(temp_file, output_file)
   ```

2. **Add progress checkpoints**: Save intermediate results periodically.

3. **Monitor worker health**: Implement heartbeat or periodic status updates.

4. **Use smaller batch sizes**: Instead of 50 seeds at once, run in smaller batches (e.g., 10 seeds).

5. **Add validation at write time**:
   ```python
   def safe_h5_write(filename, data_dict):
       """Safely write data to HDF5 with validation."""
       with h5py.File(filename, 'w') as f:
           for key, value in data_dict.items():
               f.create_dataset(key, data=value)
           # Add validation checksum
           f.attrs['checksum'] = hash(str(data_dict))
       
       # Verify write was successful
       assert is_valid_hdf5(filename), f"Failed to write valid HDF5: {filename}"
   ```

These modifications will help detect corrupted files early, provide cleaner error messages, and allow for partial recovery of results.
